MISSION VANASPATI - PROJECT OVERVIEW
================================================================================

WHAT IS THIS PROJECT?
--------------------------------------------------------------------------------
Mission Vanaspati is a web application that helps farmers and gardeners identify
plant diseases from photos. Users upload images of plant leaves, and the system
uses AI (Artificial Intelligence) to detect what disease the plant has.

Think of it like a doctor for plants - you show a picture, and it diagnoses the
problem!


PROJECT TYPE
--------------------------------------------------------------------------------
This is a FULL-STACK WEB APPLICATION, which means:
- FULL-STACK: It has both a visible website (frontend) AND a server (backend)
- WEB APPLICATION: Users access it through a web browser (Chrome, Firefox, etc.)


MAIN FEATURES
--------------------------------------------------------------------------------
1. USER AUTHENTICATION
   - Users can create accounts (Sign Up)
   - Users can login with email and password
   - Their data is private and secure

2. DISEASE DETECTION
   - Upload single plant image → Get instant diagnosis
   - Upload multiple images at once (Batch Mode) → Analyze many plants
   - Shows disease name and confidence percentage (how sure the AI is)

3. DIAGNOSIS HISTORY
   - All past diagnoses are saved in a database
   - Users can view their previous results anytime
   - Works across devices (login from phone, see history from computer)

4. REMEDY INFORMATION
   - After detecting disease, shows treatment suggestions
   - Lists organic remedies, chemical treatments, and prevention tips
   - Includes product recommendations

5. MY GARDEN
   - Users can save plants they're monitoring
   - Track plant health status over time
   - Add personal notes about each plant

6. ADMIN PANEL
   - Special users (admins) can manage the system
   - View all users, feedback, and analytics
   - Manage user permissions

7. DISEASE LIBRARY
   - Reference guide of all supported plant diseases
   - Educational content about each disease


WHO USES THIS?
--------------------------------------------------------------------------------
PRIMARY USERS:
- Farmers who need to identify crop diseases quickly
- Home gardeners caring for their plants
- Agricultural students learning about plant diseases

ADMIN USERS:
- System administrators who manage the platform
- Agricultural experts who can review and improve the system


TECHNOLOGY SIMPLE EXPLANATION
--------------------------------------------------------------------------------
Think of your application like a restaurant:

FRONTEND (React) = The dining area where customers sit
- What customers see and interact with
- Beautiful design, menus, buttons

BACKEND (FastAPI/Python) = The kitchen where food is prepared
- Does all the heavy work behind the scenes
- Processes requests, makes decisions

DATABASE (PostgreSQL) = The storage room with ingredients
- Stores all data (users, history, saved plants)
- Keeps information safe and organized

AI MODEL (PyTorch) = The chef's special recipe
- The "brain" that recognizes plant diseases
- Trained on thousands of plant images
